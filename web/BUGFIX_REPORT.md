# Отчет об исправлении багов

## Дата: 2026-01-26

### Исправленные проблемы:

#### 1. ✅ Ошибка "Rendered more hooks than during the previous render"

**Причина:**
- В `Home.tsx` условные возвраты (`if (childrenStatsError)`, `if (isLoading)`, `if (normalizedChildrenStats.length === 0)`) происходили ПОСЛЕ некоторых хуков, но ДО других хуков
- Это нарушало правила React Hooks, которые требуют, чтобы все хуки вызывались в одинаковом порядке на каждом рендере

**Исправление:**
- Перемещены все хуки (`useMemo`, `useCallback`) ДО условных возвратов
- Перемещены вычисления сытости (`saturationPercent`, `saturationColor`) ДО условных возвратов
- Все хуки теперь вызываются в строго одинаковом порядке всегда

#### 2. ✅ Условный рендеринг ChildSwitcher

**Причина:**
- `ChildSwitcher` рендерился условно в нескольких компонентах (`Approvals.tsx`, `Rewards.tsx`, `Analytics.tsx`, `Challenges.tsx`)
- Это могло вызывать проблемы, если компонент рендерился/не рендерился в зависимости от условий

**Исправление:**
- Убран условный рендеринг `ChildSwitcher` во всех компонентах
- Компонент теперь всегда рендерится, но получает пустой массив `childrenStats` если нет данных
- Добавлен флаг `isLoading` для отображения состояния загрузки

#### 3. ✅ Дублирование переменной children в Challenges.tsx

**Причина:**
- Переменная `children` объявлялась дважды: через `useChildren()` и через `useQuery()`
- Это вызывало конфликты и непредсказуемое поведение

**Исправление:**
- Удален неиспользуемый импорт `useChildren`
- Используется только один источник данных через `useQuery`

#### 4. ✅ ChildSwitcher без хуков

**Причина:**
- Компонент `ChildSwitcher` содержал хуки, что могло вызывать проблемы с порядком хуков

**Исправление:**
- Полностью переписан `ChildSwitcher` как простая функция без хуков
- Все вычисления выполняются как обычный JavaScript
- Компонент теперь полностью контролируемый (controlled component)

3. `web/src/pages/parent/Challenges.tsx`
   - Удален неиспользуемый импорт `useChildren`
   - Убран условный рендеринг `ChildSwitcher`
   - Исправлено дублирование переменной `children`

4. `web/src/pages/parent/Approvals.tsx`
   - Убран условный рендеринг `ChildSwitcher`

5. `web/src/pages/parent/Rewards.tsx`

### Что нужно сделать:

1. **Очистить кеш браузера:**
   - Mac: `Cmd + Shift + R`
   - Windows/Linux: `Ctrl + Shift + R`
   - Или через DevTools: Network → "Disable cache" → F5

2. **Перезапустить frontend:**
   ```bash
   cd web
   npm run dev
   ```

3. **Проверить консоль браузера:**
   - Откройте DevTools (F12) → Console
   - Не должно быть ошибки "Rendered more hooks than during the previous render"
   - Если ошибка осталась, проверьте Sources → `ChildSwitcher.tsx` - в файле не должно быть хуков

### Технические детали:

- Все хуки теперь вызываются в строго одинаковом порядке на каждом рендере
- `ChildSwitcher` полностью контролируемый компонент без внутренних хуков
- Все условные возвраты происходят ПОСЛЕ всех хуков
- Все вычисления, которые не являются хуками, выполняются ДО условных возвратов
