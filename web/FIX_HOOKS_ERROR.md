## Что нужно сделать:

### 1. Очистить кеш браузера

**Важно!** Браузер может использовать старую версию файла из кеша.

#### Способ 1: Жесткая перезагрузка
- **Mac**: `Cmd + Shift + R`
- **Windows/Linux**: `Ctrl + Shift + R`

#### Способ 2: Очистить кеш через DevTools
1. Откройте DevTools (F12)
2. Перейдите на вкладку **Network**
3. Включите опцию **"Disable cache"**
4. Обновите страницу (F5)

#### Способ 3: Полная очистка кеша браузера
- **Chrome/Edge**: Настройки → Конфиденциальность → Очистить данные → Кешированные изображения и файлы
- **Firefox**: Настройки → Приватность → Очистить данные → Кеш

### 2. Перезапустить frontend

```bash
cd web
npm run dev
```

### 3. Проверить в консоли браузера

Откройте DevTools (F12) → вкладка **Console** и проверьте:
- ❌ Не должно быть ошибки "Rendered more hooks than during the previous render"
- ✅ Если ошибка все еще есть, проверьте вкладку **Sources** → найдите `ChildSwitcher.tsx` и убедитесь, что код не содержит хуков

### 4. Если проблема осталась

1. **Проверьте версию файла**: В `ChildSwitcher.tsx` должна быть строка `// ВЕРСИЯ: 2.0.0`
2. **Проверьте консоль**: Откройте DevTools → Sources → найдите `ChildSwitcher.tsx` и проверьте, что в файле нет хуков
3. **Полная очистка**: Закройте все вкладки браузера, очистите кеш полностью, перезапустите браузер

## Технические детали:

- `ChildSwitcher` теперь полностью контролируемый компонент (controlled component)
- Все состояние управляется из родительских компонентов
- Компонент обернут в `React.memo` для оптимизации (но без хуков внутри)
- Все вычисления выполняются как обычный JavaScript во время рендера
